Aaron Levie (Box CEO)

AI 领域的竞争正在演变成一场关于 context(上下文)的争夺战。他指出，agent 的有效性取决于三个要素:正确的领域专业知识、对正确上下文和工具的访问权限，以及用户能够轻松交互、审查并将其纳入工作流程的能力。因此，那些能够捕获并利用最佳上下文的平台将成为赢家。Levie 认为，这也解释了为什么 applied AI layer(应用层 AI)远不仅仅是一个 LLM wrapper:它需要组织关键知识、以受治理的方式维护这些知识，并且能够随着时间推移不断改善 agent 的上下文。从长远来看，这一层将负责在不同的模型之间路由工作，利用前沿智能进行规划和编排，同时使用成本更低的模型处理大量中间任务。此外，应用层也是大部分工作流变革管理发生的地方，能够以端到端方式解决客户特定业务问题的公司将拥有最深的 护城河。详情

与 AI 协作时，最重要的部分是发现你自己的 unknowns(未知领域)，这样你才能写出更好的 prompt。Thariq 分享了他的方法论:通过使用 HTML artifacts 来帮助自己梳理在使用 Fable 时有哪些盲区。他提到，这种探索自己未知区域的方法是他近期 AIE 演讲的核心内容之一，并且特别感谢了与 Geoffrey Litt 的深入讨论，后者帮助他完善了这一思路，包括在 artifacts 中加入 quizzes(小测试)来强化发现盲点的效果。详情

Guillermo Rauch (Vercel CEO)

未来的核心在于 Agentic self-improvement(agent 自我改进)。Rauch 提出了一个框架:赋予 agent 内省过去运行记录的能力，让它自动发现低效环节、错误和冗余的 tool calls，进而生成新的 prompt 和技能。他强调，这正是为什么在将应用部署到 Vercel 时，agent observability(agent 可观测性)是内置功能:没有对运行数据的深度观察，agent 就无法实现自我迭代和优化。详情

Peter Steinberger (OpenClaw & OpenAI)

如果你觉得 codex 在 UI 设计方面表现糟糕，可以尝试一个新 prompt 技巧:要求模型 "use imagegen to re-imagine this design and implement that"(用图像生成模型重新构想这个设计并实现它)。Steinberger 发现，通过先让图像生成模型视觉化地重新构思界面，再让代码模型去实现，可以大幅提升 AI 生成前端界面的美观度和设计感。此外，他还尝试将自己的 8 万条推文喂给 Fable，让模型对他进行更精准的吐槽和互动。详情

Swyx (Latent Space / AI Engineer)

指出一个充满讽刺意味的现象:那些研究"tools for thought"(思维工具)的人花了十年时间制作各种炫酷的画布 demo，结果却被那些低对比度、设计粗糙的 CLIs(命令行界面)彻底击败。原因很简单:CLI 之所以能赢，是因为它们能切切实实地帮你完成 commodity thinking(基础思考工作)，而不仅仅是停留在演示阶段。这提醒开发者，工具的价值在于实际解决问题，而非视觉上的花哨。详情
