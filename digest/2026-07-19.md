### Anthropic Engineering: An update on recent Claude Code quality reports

Anthropic 详细复盘了近期关于 Claude 质量下降的用户反馈，确认问题源于三个独立的变更，且 API 层面完全未受影响。第一，为了降低 UI 卡顿和高延迟，他们将 Claude Code 的默认推理强度从 high 降为 medium，但这牺牲了智能水平，已于 4 月 7 日回滚。第二，清理闲置会话旧思维链的改动出现 bug，导致每轮对话都会错误清理上下文，让模型显得健忘且重复。第三，减少冗长输出的系统 prompt 指令意外损害了编码质量。这些问题叠加在一起，造成了广泛且不一致的质量衰退假象。Anthropic 强调从未故意降级模型，并已为所有订阅者重置了使用额度。[详情](https://www.anthropic.com/engineering/april-23-postmortem)

### Anthropic Engineering: Scaling Managed Agents: Decoupling the brain from the hands

Anthropic 推出了 Managed Agents，这是一项在 Claude 平台上运行的托管服务，旨在解决长周期 agent 的架构问题。核心设计理念是将 agent 的组件虚拟化，借鉴操作系统的设计哲学，将系统拆分为 session（只追加的事件日志）、harness（调用模型和工具的循环）和 sandbox（执行环境）。文章指出，早期将所有组件塞进单个容器会导致"宠物"式的基础设施困境，一旦容器卡死，不仅会丢失会话，还因为里面含有用户数据而极难调试。解决方案是将代表"大脑"的 Claude 与代表"手脚"的 sandbox 和 session 彻底解耦，让每个接口都能独立失效或替换，从而构建出能够适应未来模型进化的稳健架构。[详情](https://www.anthropic.com/engineering/managed-agents)

### Claude Blog: New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels

Claude Managed Agents 现已支持**自托管沙盒**和 MCP tunnels（研究预览版）。这意味着 agent 执行工具的沙盒环境可以完全运行在你控制的基础设施内，或者通过 Cloudflare、Daytona、Modal、Vercel 等托管提供商处理计算和隔离。agent 的编排、上下文管理和错误恢复依然留在 Anthropic 基础设施上，但代码执行、敏感文件和私有 API 调用都留在你的企业安全边界内。新增的 MCP tunnels 功能允许 agent 通过轻量级网关的单向出站连接，安全访问私有网络内部的数据库和工具，全程端到端加密，无需开放任何入站端口，极大地提升了企业级数据安全性。[详情](https://claude.com/blog/claude-managed-agents-updates)
