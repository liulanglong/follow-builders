Anthropic Engineering: An update on recent Claude Code quality reports

Anthropic 详细复盘了近一个月来用户反馈 Claude 回答质量下降的根本原因。调查发现，这是三个独立变更叠加导致的：首先，为了降低延迟，团队将 Claude Code 的默认 reasoning effort 从 high 降到了 medium，这是一个错误的取舍，已于 4 月 7 日回滚；其次，一个清理闲置 session 旧 thinking 的 bug 导致模型在每轮对话中都丢失上下文，使其显得健忘且重复，已于 4 月 10 日修复；最后，一条旨在减少冗长的 system prompt 指令与其他 prompt 变更叠加后损害了 coding 质量，已于 4 月 20 日撤销。API 和推理层从未受到影响。Anthropic 强调"我们永远不会故意降级我们的模型"，并为所有订阅者重置了 usage limits。详情

Anthropic Engineering: Scaling Managed Agents: Decoupling the brain from the hands

Anthropic 在这篇文章中介绍了 Managed Agents 的架构设计哲学：将 agent 的"大脑"（Claude 及其 harness）与"双手"（sandbox 和工具）以及"会话"（session 日志）完全解耦。这种设计借鉴了操作系统将硬件虚拟化为"进程"和"文件"的思路，使得每个组件都可以独立替换或升级而互不干扰。团队坦承最初将所有组件塞进单一容器是一个错误，这让他们陷入了"宠物"（pet）而非"牛"（cattle）的运维困境：一旦容器卡死，整个 session 就会丢失，且因为容器内含有用户数据，连调试都变得极其困难。新的 解耦架构允许 Claude 连接到客户自己的 VPC 环境，极大提升了灵活性和可调试性。详情

Claude Blog: New in Claude Managed Agents: self-hosted sandboxes and MCP tunnels

Claude Managed Agents 现在支持 self-hosted sandboxes（公开测试）和 MCP tunnels（研究预览）。这意味着 agent 执行工具的 sandbox 可以完全运行在你控制的基础设施内，敏感文件和代码库不会离开你的安全边界。你可以自带 sandbox client，也可以选择 Cloudflare（微 VM 隔离）、Daytona（全功能可组合计算机）、Modal（专为 AI 工作负载构建）或 Vercel（毫秒级启动）等受支持的提供商。此外，MCP tunnels 让 agent 能够通过一个轻量级网关安全地访问你私有网络内部的 MCP 服务器，无需开放任何入站防火墙规则或公开端点。详情
