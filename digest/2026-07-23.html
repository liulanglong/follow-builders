Claude Blog: Building intelligent apps for Apple platforms with Claude in the Foundation Models framework

Anthropic 发布了一个全新的 Swift package，使 Apple 开发者能够通过 Foundation Models framework 直接调用 Claude 来处理更复杂的工作流。Apple 的原生框架本就允许开发者通过极简代码调用设备端模型，实现快速的本机任务（如摘要提取或信息分类）。而现在，当应用需要处理多步推理、代码生成或联网搜索等复杂需求时，开发者可以无缝将请求移交给 Claude。最关键的是，因为 Apple 的框架能返回强类型的 Swift 值，Claude API 接收到的将是结构化的纯净输入，并且 Claude 处理后的结果也能直接流式传输回开发者的 SwiftUI 界面中。例如，一个日记应用可以在本机生成每日提示，然后调用 Claude 跨月度分析日记中的情感线索，从而实现端侧小模型与云端大模型的完美协同。该功能将在明天正式上线，支持 iOS 27、macOS 27 等全系新系统。详情

Training Data · Factory's Matan Grinberg: The Coming 'Dark Factory' Where Software Builds Itself

核心收获：企业最害怕在 AI 时代被单一巨头锁定，真正的护城河在于提供顶级模型性能的同时保持绝对的模块化与供应商中立。

Factory 联合创始人兼 CEO Matan Grinberg 分享了企业级 AI 编程市场的残酷竞争逻辑。当被问及与 Anthropic 的 Claude Code 或 OpenAI 的 Codex 竞争时，他一针见血地指出，企业对单一模型提供商存在极大的不信任感。他强调："所有人都从云时代的 vendor lock-in（供应商锁定）中吸取了伤疤。"当年企业被 AWS 或 Azure 用低价合同套牢，续约时面临价格暴增的教训历历在目。因此，Factory 的核心卖点在于模型独立性：如果明天出了一个更快、更便宜的模型，开发者可以立刻热插拔，而无需担心被任何 AI 实验室卡住命运的咽喉。

回顾创业初期，Grinberg 坦言在 2023 年 4 月就开始做全自动编程 agent 简直是一场荒野流浪。他苦笑着说："早了两三年，跟做错了没什么区别。"当时他劝说了 20 位顶尖人才放弃高薪加入，但无论是企业采购部门还是开发者，都对 autonomous agents 毫无准备。但正是这两年的沉淀，让 Factory 深刻理解了企业的核心诉求：不仅需要强大的自动化能力，更需要确保所有的代码生成和工作流产物都留在客户自己的代码库中。Factory 不搞任何知识垄断，这种极致的信任机制最终成为了他们拿下大型企业客户的关键筹码。详情
